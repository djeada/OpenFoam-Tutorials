## Problem Description

The lid-driven cavity is one of the most widely studied benchmark problems in computational fluid dynamics. A square cavity is filled with an incompressible fluid. All walls are stationary except the top wall (the "lid"), which moves at a constant horizontal velocity. The moving lid drags the fluid beneath it through viscous shear, generating a recirculating flow inside the cavity.

Despite its geometric simplicity, the lid-driven cavity produces rich flow physics: a primary recirculating vortex dominates the cavity interior, and depending on the Reynolds number, secondary and even tertiary corner vortices appear near the bottom corners. This makes it an ideal first case for validating CFD solvers and learning the fundamentals of setting up a simulation.

```
         U = (1, 0, 0)  →→→→→→→→→→→→→→→→→→
        ┌──────────────────────────────────┐
        │                                  │
        │      ↗  →  →  →  →  ↘           │
        │      ↑                ↓          │
        │      ↑   Primary     ↓          │
        │      ↑    Vortex     ↓          │
        │      ↑               ↓          │
        │      ↖  ←  ←  ←  ↙             │
 noSlip │                                  │ noSlip
        │                                  │
        │  ↻ secondary          secondary ↺│
        │    corner               corner   │
        │    vortex               vortex   │
        └──────────────────────────────────┘
                  noSlip (fixedWalls)
```

At the Reynolds number used in this tutorial ($Re = 100$), the flow is laminar and steady-state is reached quickly. The primary vortex fills most of the cavity, with its center shifted slightly toward the upper-right quadrant. At higher Reynolds numbers ($Re > 1000$), the corner vortices become more prominent and the flow eventually transitions to unsteady behavior.

## Physics and Theory

### Governing Equations

This case solves the **incompressible Navier-Stokes equations** for a Newtonian fluid:

**Continuity (mass conservation):**

$$\nabla \cdot U = 0$$

**Momentum conservation:**

$$\partial U / \partial t + (U \cdot \nabla)U = -\nabla p / \rho + \nu \nabla^2 U$$

where `U` is the velocity vector, `p` is the kinematic pressure (pressure divided by density), and `ν` is the kinematic viscosity.

Since icoFoam is an incompressible solver, density does not appear explicitly. The pressure field `p` has dimensions of m$^2$/s$^2$ (kinematic pressure).

### Reynolds Number

The Reynolds number characterizes the ratio of inertial to viscous forces:

$$Re = U \times L / \nu$$

For this case:

| Parameter | Symbol | Value |
|---|---|---|
| Lid velocity | U | 1 m/s |
| Cavity width | L | 0.1 m |
| Kinematic viscosity | $\nu$ | 0.001 m$^2$/s |

$$Re = 1 \times 0.1 / 0.001 = 100$$

At $Re = 100$, the flow is well within the laminar regime. The primary vortex forms quickly and the solution reaches a quasi-steady state within the 0.5 s simulation window.

### Expected Flow Features

- **Primary vortex**: A large clockwise recirculation filling most of the cavity. At $Re = 100$, the vortex center is located approximately at ($x/L \approx 0.62$, $y/L \approx 0.74$) based on the Ghia et al. (1982) benchmark data.
- **Bottom corner vortices**: Small, weak counter-rotating vortices in the bottom-left and bottom-right corners. These become increasingly prominent at higher Reynolds numbers.
- **Boundary layers**: Thin regions of high velocity gradient along all walls, particularly beneath the moving lid.

## Case Structure

```
01_lid_driven_cavity/
├── README.md                          ← This file
├── 0/                                 ← Initial and boundary conditions (t = 0)
│   ├── U                              ← Velocity field [m/s]
│   └── p                              ← Kinematic pressure field [m²/s²]
├── constant/                          ← Physical properties and mesh
│   ├── transportProperties            ← Fluid properties (viscosity)
│   └── polyMesh/                      ← Mesh data (generated by blockMesh)
│       ├── boundary
│       ├── faces
│       ├── neighbour
│       ├── owner
│       └── points
└── system/                            ← Simulation control
    ├── blockMeshDict                  ← Mesh generation parameters
    ├── controlDict                    ← Run control (time, output, solver)
    ├── fvSchemes                      ← Discretization schemes
    └── fvSolution                     ← Linear solver and algorithm settings
```

The standard OpenFOAM "three-directory" structure is used:

- **`0/`** contains initial conditions and boundary conditions for each field variable.
- **`constant/`** holds time-invariant data: physical properties and the mesh.
- **`system/`** contains all simulation control parameters.

## Mesh Details

The mesh is generated by `blockMesh` from the dictionary at `system/blockMeshDict`.

### Domain Geometry

The `convertToMeters` factor is **0.1**, so the unit-cube vertices in the dictionary map to physical coordinates:

| Property | Value |
|---|---|
| Domain size (x $\times$ y $\times$ z) | 0.1 m $\times$ 0.1 m $\times$ 0.01 m |
| Cell count | 20 $\times$ 20 $\times$ 1 = 400 cells |
| Cell type | Hexahedral (structured) |
| Grading | Uniform (simpleGrading 1 1 1) |
| Cell size ($\Delta x$ $\times$ $\Delta y$) | 0.005 m $\times$ 0.005 m |

The single-cell thickness in z and the `empty` boundary condition on the front/back faces make this a **2D simulation**.

## Boundary Conditions

All boundary conditions are defined in `0/U` and `0/p`.

### Summary Table

| Boundary | Patch Type | Velocity (`U`) | Pressure (`p`) | Physical Meaning |
|---|---|---|---|---|
| `movingWall` | `wall` | `fixedValue` uniform (1 0 0) | `zeroGradient` | Lid moving at 1 m/s in x |
| `fixedWalls` | `wall` | `noSlip` | `zeroGradient` | Stationary walls (bottom, left, right) |
| `frontAndBack` | `empty` | `empty` | `empty` | 2D constraint (z-normal faces) |

### Notes on Boundary Conditions

- **`fixedValue`**: Sets velocity directly. The lid moves at exactly 1 m/s in the positive x-direction.
- **`noSlip`**: A shorthand for `fixedValue uniform (0 0 0)`. No fluid motion at stationary walls.
- **`zeroGradient`**: The normal gradient of pressure is zero at walls. This is the standard Neumann condition for pressure in incompressible wall-bounded flows.
- **`empty`**: Tells OpenFOAM to ignore these faces for calculation, enabling a 2D simulation in the x-y plane.
- **Pressure reference**: Since the cavity has no inlet or outlet, pressure is only defined up to a constant. The `pRefCell 0` and `pRefValue 0` settings in `fvSolution` pin cell 0 to p = 0 to anchor the pressure field.

> **Further reading:** [Boundary conditions in OpenFOAM](https://github.com/djeada/OpenFoam-Tutorials/blob/main/notes/05_boundary_conditions.md)

## Solver Configuration

### controlDict — Run Control

| Setting | Value | Meaning |
|---|---|---|
| `application` | `icoFoam` | Incompressible laminar transient solver |
| `startTime` | `0` | Simulation begins at t = 0 s |
| `endTime` | `0.5` | Simulation ends at t = 0.5 s |
| `deltaT` | `0.001` | Fixed time step of 1 ms |
| `writeControl` | `timeStep` | Output triggered by time step count |
| `writeInterval` | `1` | Write output every time step |
| `writeFormat` | `ascii` | Human-readable output files |
| `purgeWrite` | `0` | Keep all time directories (no pruning) |

With `deltaT = 0.001` and `endTime = 0.5`, the solver runs for **500 time steps**. Because `writeInterval` is set to 1, every single time step is saved. For production runs you may want to increase `writeInterval` (e.g., to 20 or 50) to reduce disk usage.

**CFL number estimate:** With $\Delta x$ = 0.005 m and U_max = 1 m/s:

$$CFL = U \times \Delta t / \Delta x = 1 \times 0.001 / 0.005 = 0.2$$

A CFL of 0.2 is well below the stability limit of 1.0, ensuring a stable explicit time integration.

### fvSchemes — Discretization Schemes

| Category | Scheme | Description |
|---|---|---|
| `ddtSchemes` | `Euler` | First-order implicit time discretization |
| `gradSchemes` (default) | `Gauss linear` | Second-order central differencing for gradients |
| `gradSchemes` (grad(U)) | `cellLimited Gauss linear 1` | Gradient limiter for velocity to prevent oscillations |
| `divSchemes` (div(phi,U)) | `Gauss linearUpwind grad(U)` | Second-order upwind for convection term |
| `laplacianSchemes` | `Gauss linear corrected` | Second-order with non-orthogonal correction for diffusion |
| `interpolationSchemes` | `linear` | Linear interpolation from cell centers to faces |
| `snGradSchemes` | `corrected` | Non-orthogonal corrected surface-normal gradients |

### fvSolution — Linear Solvers and PISO

**Pressure solver (`p`):**

| Setting | Value |
|---|---|
| Solver | `PCG` (Preconditioned Conjugate Gradient) |
| Preconditioner | `DIC` (Diagonal Incomplete Cholesky) |
| Tolerance | 1$\times$10$^{-6}$ |
| Relative tolerance | 0.05 (0 for final corrector) |

**Velocity solver (`U`):**

| Setting | Value |
|---|---|
| Solver | `smoothSolver` |
| Smoother | `symGaussSeidel` (Symmetric Gauss-Seidel) |
| Tolerance | 1$\times$10$^{-5}$ |
| Relative tolerance | 0 |

**PISO algorithm settings:**

| Setting | Value | Meaning |
|---|---|---|
| `nCorrectors` | `2` | Number of pressure-velocity correction loops per time step |
| `nNonOrthogonalCorrectors` | `0` | No non-orthogonal correction (mesh is orthogonal) |
| `pRefCell` | `0` | Reference pressure cell index |
| `pRefValue` | `0` | Reference pressure value (anchors the pressure field) |

> **Further reading:** [Linear solvers in OpenFOAM](https://github.com/djeada/OpenFoam-Tutorials/blob/main/notes/09_linear_solvers.md)

## How to Run

### Prerequisites

- OpenFOAM 6 (or compatible version) installed and environment sourced
- ParaView for post-processing (optional)

### Step-by-Step Commands

Navigate to this project directory, then run:

```bash
# 1. Generate the mesh from blockMeshDict
blockMesh

# 2. Verify mesh quality (check for errors, non-orthogonality, skewness)
checkMesh

# 3. Run the solver
icoFoam

# 4. Convert results to VTK format for ParaView
foamToVTK

# 5. Open in ParaView for visualization
paraview
```

**Or as a single pipeline:**

```bash
blockMesh && checkMesh && icoFoam && foamToVTK
```

### What to Expect During the Run

- **`blockMesh`** prints the vertex coordinates, block definitions, and patch information. Look for `End` with no errors.
- **`checkMesh`** reports mesh statistics. For this case you should see: 400 hexahedral cells, 0 failed checks, max non-orthogonality = 0 (perfectly orthogonal mesh).
- **`icoFoam`** prints residuals for `p` and `U` at each time step. Initial residuals will be large but should decrease over time. The solver runs 500 time steps and takes a few seconds on modern hardware.
- Time directories (`0.001/`, `0.002/`, ..., `0.5/`) are created containing the velocity and pressure fields at each step.

### Cleaning Up

To reset the case and remove all generated data:

```bash
foamCleanCase
```

Or manually:

```bash
rm -rf 0.* [1-9]* VTK/ constant/polyMesh
```

## Expected Results

### Flow Field at Steady State (t $\approx$ 0.5 s)

At $Re = 100$, the flow reaches a quasi-steady state well within the 0.5 s simulation window. You should observe:

- **Primary vortex**: A large clockwise recirculation filling most of the cavity. The vortex center is located near (x = 0.062 m, y = 0.074 m), approximately (x/L = 0.62, y/L = 0.74).
- **Maximum velocity**: Close to 1 m/s near the top wall (the lid).
- **Corner vortices**: Very small and weak at $Re = 100$; they become visible at higher Reynolds numbers.
- **Pressure field**: Low pressure near the vortex center, higher pressure at the upper corners where the flow decelerates.

### Vertical Centerline Velocity Profile (Ux along x = 0.05 m)

The horizontal velocity component along a vertical line through the center of the cavity has a characteristic S-shaped profile:

```
  y/L
  1.0 ┤ ←── Ux = 1.0 (lid velocity)
      │                    ·
  0.9 ┤                  ·
      │                ·
  0.8 ┤              ·
      │            ·
  0.7 ┤          ·
      │         ·
  0.6 ┤        ·
      │        ·
  0.5 ┤       ·
      │      ·
  0.4 ┤     ·
      │    ·
  0.3 ┤   ·
      │  ·
  0.2 ┤ ·
      │·
  0.1 ┤·
      │ ·
  0.0 ┤  ←── Ux = 0 (no-slip at bottom wall)
      └──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──→ Ux
       -0.4       0.0        0.5     1.0

  The velocity is ~1.0 at the top (lid), drops rapidly through
  the boundary layer, is slightly negative in the lower half
  (return flow of the primary vortex), and reaches 0 at the bottom.
```

### Horizontal Centerline Velocity Profile (Uy along y = 0.05 m)

The vertical velocity along the horizontal centerline shows:
- Positive Uy near the right wall (upward flow)
- Negative Uy near the left wall (downward flow)
- Zero at the center of the vortex

## Visualization Guide (ParaView)

After running `foamToVTK`, open the VTK files in ParaView:

```bash
paraview VTK/cavity_..vtk
```

### Recommended Visualizations

1. **Velocity magnitude contour:**
   - Select the VTK data → Apply
   - In the toolbar dropdown, change `Solid Color` to `U` (magnitude)
   - Use the `Rescale to Data Range` button for proper color mapping

2. **Streamlines:**
   - Filters → Alphabetical → Stream Tracer
   - Set seed type to `Point Cloud` or `Line`
   - Place the seed line along the vertical centerline
   - Streamlines will reveal the primary vortex structure

3. **Pressure contour:**
   - Color by `p` to see the pressure field
   - Low pressure at the vortex center, high pressure at stagnation points

4. **Vector plot:**
   - Filters → Alphabetical → Glyph
   - Set Scale Mode to `vector` and choose `U`
   - Reduce the scale factor to avoid overcrowding

5. **Centerline data extraction:**
   - Filters → Alphabetical → Plot Over Line
   - Set the line from (0.05, 0, 0.005) to (0.05, 0.1, 0.005) for a vertical centerline
   - Plot Ux vs. y to compare with benchmark data

## Exercises and Experiments

These exercises build progressively on the base case. Try them in order:

### 1. Change the Reynolds Number

Modify `constant/transportProperties` to change `ν` and observe different flow regimes:

| $\nu$ (m$^2$/s) | Re | Expected Behavior |
|---|---|---|
| 0.01 | 10 | Very viscous, weak vortex, slow approach to steady state |
| 0.001 | 100 | ← Current case. Clear primary vortex |
| 0.0001 | 1000 | Stronger vortex, visible corner vortices, longer to converge |
| 0.00001 | 10000 | Unsteady flow, requires finer mesh and smaller time step |

> **Note:** At $Re > 1000$, you may need to reduce `deltaT` and refine the mesh to maintain stability and accuracy. See [CFL number](https://github.com/djeada/OpenFoam-Tutorials/blob/main/notes/08_cfl_number.md).

### 2. Increase Mesh Resolution

Edit `system/blockMeshDict` and change the cell count from `(20 20 1)` to `(40 40 1)`:

```
hex (0 1 2 3 4 5 6 7) (40 40 1) simpleGrading (1 1 1)
```

This quadruples the cell count to 1600. Compare:
- Vortex center location — does it shift?
- Velocity profiles — are they smoother?
- Residual convergence — is it faster or slower?

### 3. Apply Mesh Grading

For better resolution near the walls without increasing cell count everywhere, try:

```
hex (0 1 2 3 4 5 6 7) (20 20 1) simpleGrading (2 2 1)
```

This concentrates cells near the high-gradient regions at the walls.

### 4. Compare with Ghia et al. (1982) Benchmark

Extract the vertical centerline velocity profile and compare with the tabulated data from Ghia, Ghia, and Shin (1982). Key reference values for $Re = 100$:

| y/L | Ux (Ghia et al.) |
|---|---|
| 1.0000 | 1.00000 |
| 0.9766 | 0.84123 |
| 0.9688 | 0.78871 |
| 0.9609 | 0.73722 |
| 0.5000 | -0.06080 |
| 0.0000 | 0.00000 |

### 5. Experiment with Time Stepping

- Double `deltaT` to 0.002 (CFL $\approx$ 0.4): Should still be stable.
- Try `deltaT = 0.005` (CFL $\approx$ 1.0): At the stability boundary.
- Try `deltaT = 0.01` (CFL $\approx$ 2.0): Likely diverges — observe the behavior.

### 6. Change the Write Interval

The current `writeInterval 1` saves every time step (500 directories). Change it to write less frequently:

```
writeInterval   50;    // Save every 50 steps → 10 output directories
```

### 7. Try a Different Solver

Once comfortable with icoFoam, try `pisoFoam` (supports turbulence models) on the same case to explore turbulent cavity flow at higher Re.
